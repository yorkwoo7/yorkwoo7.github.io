Visual Component 4.7 Blender渲染插件安装

Visual Components（简称VC），是一款在智能制造、数字化工厂和数字孪生领域广泛使用的3D制造仿真软件。它以直观的界面和对多种CAD文件格式的支持，为用户提供了轻松设计和集成数字仿真环境的可能。

Blender，作为一款功能全面的免费开源3D创作套件，不仅支持建模、渲染、动画制作等3D管线的各个环节，而且在电影工业中也得到了广泛应用。

通过将VC与Blender结合，用户可以直接在Blender中操作和渲染VC的3D模型，实现更为逼真的视觉效果。以下是渲染前后的对比图：

  • 渲染后的视觉效果

通过大量时间的投入，甚至可以达到下图所示的高质量效果。

实际上，VC支持Blender插件化，这使得两者的结合更加紧密，无需导入导出模型，使用起来更为便捷。然而，在安装过程中，我遇到了一些问题，尽管最终问题得以解决，但这个过程耗费了我近4个小时。特此记录下这个经验，希望对撞上相同问题的人有所帮助。

安装步骤：

  1. 安装Visual Components 4.7：这是一个Autodesk的付费软件，请自行准备。

  2. 安装Blender：建议使用长期支持版本（LTS）。安装过程简单，按照默认选项直至完成。若遇到问题，建议自行搜索解决方案或使用我提供的下载链接:

  3. 准备Blender插件：请使用与Blender版本相匹配的插件。如果你不想自己寻找，可以使用我提供的下载链接:

安装Blender插件就是本文重点了。

​ 首先我们参考了官方文档

结果官方文档是错的！安装之后，VC里面始终找不到Render菜单。

百度之，结果资料极少，看着日期很新，但是操作明显不对，根本没有这些菜单和工具栏。

继续各种搜索，各种尝试。

因为此前从没有安装过，心里完全没有谱。甚至还怀疑起会不会是VC、Blender、Blender Plugin三者软件版本的配套关系有问题。

得用啊，不死心，反复下载不同的版本，尝试在不同的目录进行安装和测试。一些安装包还只在外网才有，下载困难，好一通折腾。

但有时候不是努力就有效果的。几个小时过去，翻遍了网上的资料，无果。

就连万能的OpenGPT也说起了车轱辘话，没了主意。

其实已经快放弃了，直到准备保存一个模型，软件会自动打开模型目录，我上翻一层。

发现这儿有一个My Commands目录！，尝试把Blender Plugins放进去。

终于看到了Blender的选项！

我服了写官方文档的老6，我现在明白了，你想写的其实是把插件文件放进 我的文档 下的 Visual Components 目录下的 My Commands 下吧？！结果写成了绝对路径，忽略了各自的电脑这个路径的变化。

所以问题其实就是出在Blender插件需要放到正确路径内。但是全网没有人说对这个 变量。

顺便写一个问题2

指定渲染路径时，提示不能选择输出的文件。

搜索发现不少人在问。

解决办法：在你想要输出的目标，手工任置一个[文件名].png，再重新指定渲染路径为这个文件。完成后，这个文件就可以删除了，输出目录也成功设定了。
